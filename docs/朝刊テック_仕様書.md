# 朝刊テック — AWS + IT ニュースまとめアプリ 仕様書

> このファイルは **Claude Code（VS Code）に渡して実装してもらうための指示書** です。
> 同じ `docs/` フォルダにある `morning_tech_briefing.html` が **完成イメージ（デザイン見本）** です。
> まずこの md と HTML の両方を Claude Code に読ませてください。

---

## 1. このアプリで実現したいこと

毎朝、AWS と IT 業界のニュースを自動で集め、**自分の興味トピックで絞り込まれた状態**で読みたい。
さらに、興味に合う新着が出たら通知を届け、**毎朝のまとめ**を自動で生成したい。

ゴールを一言でいうと、**自分専用の「毎朝のITニュース秘書」**です。

利用者は IT 企業（中小）のブログ担当者で、AWS の新機能をいち早く把握して
ブログ記事のネタ集めに使います。

## 2. 完成イメージ（必ず最初に確認）

- `morning_tech_briefing.html` をブラウザで開くと、目指すべき UI が動きます。
- **デザイン・画面構成・配色・文言は、この HTML を基準にしてください。**
- HTML はサンプルデータで動く「見た目だけの試作」です。これを土台に、
  実際のデータ取得・まとめ自動生成・通知を実装するのがこのプロジェクトの目的です。

### 画面構成（HTML と同じ4タブ）
1. **ホーム** … 興味トピックで絞り込んだ新着フィード。各記事に「なぜ表示したか（興味タグ）」を表示
2. **まとめ** … その日のニュースを 3 トピック程度に整理し、各トピックに短い要約を付けた「今朝のまとめ」
3. **通知** … 興味に合う新着・まとめ完成の通知一覧
4. **設定** … まとめの配信時刻 / 通知するトピック / ニュース取得元のオンオフ

## 4. 技術構成（おすすめ）

中小企業が無理なく運用できる、シンプルで低コストな構成を提案します。
Claude Code には、まず **最も簡単に動く構成** から作ってもらってください。

### 段階A：まずPC/サーバーで動くWebアプリ版（ここがスタート地点）
```
[定期実行] RSS取得スクリプト
   ↓ 記事を保存（SQLite など軽量DBで十分）
[まとめ生成] 取得記事を興味トピックで分類 + 要約生成（LLM API）
   ↓
[Webアプリ] スマホブラウザで開ける画面（HTMLのデザインを流用）
```
- フロントエンド：HTML/CSS/JS（`morning_tech_briefing.html` をそのまま土台にする）
- バックエンド：Python（FastAPI など）または Node.js のどちらでも可
- RSS取得：Python なら `feedparser`、Node なら `rss-parser`
- データ保存：SQLite（最初はこれで十分。ファイル1つで動く）
- まとめ生成：Anthropic API（Claude）でトピック分類と要約文生成
- 定期実行：cron / タスクスケジューラ / クラウドの定期実行

> **重要：CORSの注意**
> RSS取得は必ず**サーバー側（バックエンド）で行う**こと。
> ブラウザの JavaScript から直接 RSS を取りに行くと CORS でブロックされます。
> フロントは、自分のバックエンドが整形したデータだけを受け取る構成にしてください。

### 段階B：通知付き・常時稼働版（MVPが動いた後）

**全体方針：完全無料（追加費用0円）で組む。**

- **定期実行**：**GitHub Actions の定時実行（cron）** を採用。無料枠で毎朝1回の取得を回せる。
  自分のPCを点けっぱなしにしなくてよい（自分のPC＋タスクスケジューラだとPC起動中しか動かない）。
- **スマホアプリ化**：**PWA**（ホーム画面に追加できるWebアプリ）を採用。**費用0円**。
  iPhoneでも Safari →「ホーム画面に追加」でアプリのように起動できる。
  ネイティブアプリ（App Store配信）は Apple Developer 年額 約15,000円＋Mac必須のため見送り。
- **通知**：iOS は **16.4以降でPWAのWeb Pushに対応**。これで「自作アプリの通知」のように出せる（無料）。
  - まず確実に受け取りたい場合の代替：**Discord / メール**（既存アプリのネイティブ通知で届く・無料・設定が簡単）。
  - 注意：**LINE Notify は2025年3月末でサービス終了**。使うなら LINE Messaging API（無料枠あり・設定はやや手間）。

## 5. ニュース取得元（実際のRSS URL）

下記は実在する公式フィードです。**Claude Code は実装前にURLが有効か必ず確認**してください
（URL末尾に `/feed/` を付ける形式が多い）。

| 取得元 | RSS URL | 言語 | 備考 |
|---|---|---|---|
| AWS What's New（新機能速報） | `https://aws.amazon.com/about-aws/whats-new/recent/feed/` | 英 | 最も速報性が高い |
| AWS News Blog（英語公式） | `https://aws.amazon.com/blogs/aws/feed/` | 英 | 一次情報が早い |
| AWS 公式ブログ（日本語） | `https://aws.amazon.com/jp/blogs/news/feed/` | 日 | 翻訳不要 |
| DevelopersIO（クラスメソッド） | `https://dev.classmethod.jp/feed/` | 日 | やってみた系が豊富 |
| Publickey | `https://www.publickey1.jp/atom.xml` | 日 | IT全般 |
| 日経クロステック | （要確認） | 日 | **一部は有料会員向けで全文RSS不可。タイトル/リンクのみ取得にとどめる** |

- 各AWSブログは、ブログトップURLの末尾に `/feed/` を付ければRSSになります
  （例：Database Blog, Developer Tools Blog なども同様に追加可能）。
- 取得元は設定画面でオンオフできるよう、**コードに直書きの一覧データとして管理**してください。
- 有料サイト（日経など）は要約に注意し、取得はRSSで公開されている範囲（タイトル・要約・リンク）に限定。

## 6. 興味トピックの分類ルール

記事を以下のトピックに自動分類します（1記事が複数トピックに属してOK）。

- **生成AI**：Bedrock, Claude, GPT, LLM, RAG, AIエージェント など
- **クラウド/インフラ**：EC2, S3, VPC, 移行, サーバーレス など
- **セキュリティ**：GuardDuty, WAF, IAM, 脆弱性, ランサムウェア など
- **データ分析**：OpenSearch, Redshift, データ基盤, BI など
- **開発ツール**：CDK, Amplify, コーディングエージェント, CI/CD など

分類方法は2案。まずは簡単な方から：
1. **キーワードマッチ**（簡単・無料）：記事タイトル/本文に上記の語が含まれるかで判定
2. **LLM分類**（精度高い）：Claude に記事を渡して、どのトピックかを判定させる

## 7. 「今朝のまとめ」生成の仕様

毎朝（設定時刻、例 7:00）に、過去24時間の新着記事から「まとめ」を生成する。

1. 過去24時間の記事を取得
2. 興味トピックごとにグループ分け
3. 各グループの要約を作る
4. HTMLの「まとめ」画面のレイアウトに流し込む

### 採用方針：まずは「抽出型」（無料・全自動）

コストをかけない方針のため、**ステップ5の要約は当面「抽出型」で実装する**。

- 各記事の RSS 概要文（`summary` / `description`）や本文の冒頭1〜2文を抜き出して、
  トピックごとに数件並べる。**LLMは使わない＝追加費用0円・完全自動**。
- トピック見出し＋記事の抜粋リスト、という形でHTMLの「まとめ」画面に流し込む。
- 弱点：文章を練った要約にはならない（あくまで抜粋集）。質より「無料・自動」を優先する判断。

### 将来の差し替え候補（質を上げたくなったら）

要約ロジックは**差し替え可能な形**で作っておく（入力＝記事リスト、出力＝要約テキスト）。

- **B案：ChatGPTに貼る半自動** … アプリが「貼り付け用テキスト」を生成 → 利用者が手持ちの
  ChatGPTサブスクに貼って要約を得る。追加費用0円・質は最高だが朝に手作業が要る。
  （注意：ChatGPTの月額サブスクに API 利用は含まれない。自動化には別途 API 課金が必要）
- **C案：API で全自動要約** … OpenAI API もしくは Claude API に記事一覧を渡して2〜3文要約。
  月数百円程度。ClaudeでもOpenAIでも実装はほぼ同じで差し替え容易。
  > 要約プロンプトには「IT企業のブログ担当者向けに、記事ネタとしての価値を一言添えて」と
  > 指示すると、HTMLの見本と同じトーンになる。

## 8. 実装ステップ（Claude Code への進め方）

一気に作らせず、段階的に。各ステップで動作確認してください。

1. **ステップ1**：RSS取得スクリプトを作り、5〜6個のフィードから記事一覧を取得してコンソールに出力
2. **ステップ2**：取得結果をSQLiteに保存（重複は登録しない）
3. **ステップ3**：記事をトピック分類（まずキーワードマッチ）
4. **ステップ4**：`morning_tech_briefing.html` を土台に、バックエンドの実データを表示するWebアプリにする
5. **ステップ5**：「今朝のまとめ」自動生成を実装（Claude API）
6. **ステップ6**：定期実行（cron等）で毎朝自動更新
7. **ステップ7**：通知（メール or LINE）を追加
8. **ステップ8**：PWA化してスマホのホーム画面に追加できるように

## 9. 制約・注意点（正直な前提）

- **本物のプッシュ通知・毎朝の自動実行**には、24時間動く裏側（サーバー/定期実行）が必要。
  ローカルPCだけだと、PCを起動している時しか動かない。
- **日経など有料サイト**は全文RSSを返さないことが多く、タイトル・リンクのみに留まる。
- **LLMでの要約・分類はAPI利用料**が少しかかる（1日数十記事程度なら少額）。
- ネイティブのスマホアプリ（App Store / Google Play 配信）は、まずPWAで体験を作ってから検討。
